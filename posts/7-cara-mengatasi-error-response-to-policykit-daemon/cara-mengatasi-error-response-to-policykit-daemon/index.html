Cara Mengatasi Error Response To PolicyKit Daemon

Polkit (sebelumnya PolicyKit ) adalah komponen untuk mengontrol hak istimewa seluruh sistem dalam sistem operasi mirip Unix.
September 8, 2023 PolkitDaemon 178 words 1 min

Pesan kesalahan ini saya dapatkan ketika menjalankan perintah $ pkexec ls yang menunjukan keluaran sebagai berikut.

==== AUTHENTICATING FOR org.freedesktop.policykit.exec ====
Authentication is needed to run `/usr/bin/ls' as the super user
Authenticating as: anggiramadyansyah
Password:
polkit-agent-helper-1: error response to PolicyKit daemon: GDBus.Error:org.freedesktop.PolicyKit1.Error.Failed: No session for cookie
==== AUTHENTICATION FAILED ====
Error executing command as another user: Not authorized

This incident has been reported.

Sebelumnya, saya memang tidak menggunakan polkit berbasis antar muka pengguna grafis seperti:

  1. Polkit-gnome
  2. Xfce-polkit
  3. Mate-polkit
  4. Dll.

Untuk mengatasi masalah ini kita perlu tahu ID proses; PID dari shell yang sedang kita gunakan.

Untuk itu kita perlu perintah dibawah ini.

$ echo $$
2354

Maka ID proses kita saat ini adalah 2354.

Setelah itu kita harus membuka sebuah terminal kembali dengan isi perintah seperti ini.

$ pkttyagent --process 2354

Maka keluaran dari perintah di atas adalah sebagai berikut.

$ pkttyagent --process 2354                                                                                                                                                                                         
==== AUTHENTICATING FOR org.freedesktop.policykit.exec ====
Authentication is needed to run `/usr/bin/ls' as the super user
Authenticating as: anggiramadyansyah
Password:
==== AUTHENTICATION COMPLETE ====

Setelahnya, silahkan lakukan yang teman-teman inginkan dengan pkexec pada terminal pertama.
