Cara Sederhana Mengatasi Error Response To Policy Kit Daemon Pada Arch Linux
________________________________________________________________________________

Pesan kesalahan ini saya dapatkan ketika ingin melihat konten direktori dengan 
hak akses administrator yang menggunakan perintah :

  $ pkexec ls 

Namun keluaran terminal menunjukan ini :

  ==== AUTHENTICATING FOR org.freedesktop.policykit.exec ====
  Authentication is needed to run `/usr/bin/ls' as the super user
  Authenticating as: $HOME
  Password: **********
  polkit-agent-helper-1: error response to PolicyKit daemon: GDBus.Error:
  org.freedesktop.PolicyKit1.Error.Failed: No session for cookie
  ==== AUTHENTICATION FAILED ====
  Error executing command as another user: Not authorized
  This incident has been reported.

Apakah saya salah memasukan kata sandi? Tidak! Tetapi untuk mengatasi masalah 
seperti di atas, kita perlu mengetahui terlebih dahulu PID [1] pada terminal saat 
ini dengan perintah :

  $ echo $$
  2232

PID ditunjukan dengan angka 2232. Setelahnya, kita harus membuka sebuah terminal
baru dan dilanjutkan dengan :

  $ pkttyagent --process 2232

Yang mana perintah diatas adalah skenario suatu proses membutuhkan hak akses 
tambahan dan otentikasi harus dilakukan melalui terminal yang terkait dengan 
proses tersebut, nantinya keluaran akan menunjukan seperti ini :

  ==== AUTHENTICATING FOR org.freedesktop.policykit.exec ====
  Authentication is needed to run `/usr/bin/ls' as the super user
  Authenticating as: $HOME
  Password: **********
  ==== AUTHENTICATION COMPLETE ====

Keluaran tersebut menandakan bahwa proses autentikasi telah berhasil.
