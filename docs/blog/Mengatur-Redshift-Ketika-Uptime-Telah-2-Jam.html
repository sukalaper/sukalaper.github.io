Mengatur Redshift Ketika Uptime Telah 2 Jam
________________________________________________________________________________

- Konfigurasi Perangkat .................................................. [1.0]
- Referensi .............................................................. [2.0]

Redshift [1] adalah sebuah aplikasi yang digunakan untuk mengatur suhu warna 
layar komputer agar lebih nyaman untuk dilihat pada malam hari. Aplikasi ini 
mengubah warna layar menjadi lebih hangat saat malam hari untuk mengurangi 
ketegangan mata dan membantu mengurangi gangguan tidur yang dapat disebabkan 
oleh cahaya biru yang dipancarkan oleh layar.

Meskipun terdapat kalimat "pada malam hari" saya ingin mengatur temperatur 
warna ketika uptime [2] pada mesin saya telah mencapai 2 jam, hal ini di 
karenakan mata saya lebih cepat lelah ketika berada di depan komputer dalam 
jangka waktu lama.

Temperatur warna yang saya gunakan adalah rentang 3700 - 4000 [3], ini akan 
menghasilkan warna kuning yang berfungsi sebagai "mode baca", 


[1.0] Konfigurasi Perangkat
________________________________________________________________________________

Pada Arch Linux [4] Redshift dapat di pasang dengan perintah:

  $ sudo pacman -S redshift

Untuk memastikan Redshift berfungsi, gunakan perintah:

  $ redshift -O 3700

Maka warna temperatur pada layar komputer teman-teman akan berubah menjadi 
kuning. Lalu bagaimana jika saya ingin Redshift hanya aktif jika uptime saya
telah mencapai 2 jam?

  1. Buat sebuah skrip shell dengan nama "start-redshift.sh" yang akan memeriksa 
     uptime sistem dan kemudian menjalankan Redshift jika kondisi terpenuhi:

     #!/bin/bash

     # uptime dalam detik (2 jam = 7200 detik)
     UPTIME_THRESHOLD=7200
     # dapatkan uptime sistem dalam detik
     uptime_seconds=$(awk '{print $1}' /proc/uptime | cut -d '.' -f 1)
     # Periksa apakah uptime lebih dari threshold
     if [ "$uptime_seconds" -ge "$UPTIME_THRESHOLD" ]; then
        # mulai Redshift jika kondisi terpenuhi
        redshift -O 3700 &
     fi

  2. Simpan skrip tersebut pada /usr/local/bin/ dan berikan izin eksekusi 
     dengan perintah:

     $ sudo chmod +x /usr/local/bin/start-redshift.sh

  3. Buat unit file Systemd [5] untuk menjalankan skrip ini secara otomatis pada 
     saat boot atau pada interval tertentu. Misalnya, 
     buat file /etc/systemd/system/redshift-delay.service: 

     [Unit]
     Description=Start Redshift after 2 hours uptime
     After=network.target

     [Service]
     ExecStart=/usr/local/bin/start-redshift.sh
     Type=oneshot

     [Install]
     WantedBy=default.target

     Unit file ini akan menjalankan skrip setelah network.target tercapai. 

  4. Buat timer Systemd untuk menjalankan skrip pada interval tertentu, 
     dengan nama /etc/systemd/system/redshift-delay.timer 
     dengan isi sebagai berikut:

     [Unit]
     Description=Timer for starting Redshift after uptime

     [Timer]
     OnBootSec=2h
     Unit=redshift-delay.service

     [Install]
     WantedBy=timers.target

     Unit file ini akan menunggu 2 jam setelah boot sebelum menjalankan 
     redshift-delay.service.

  5. Aktifkan dan mulai timer pada unit Systemd yang telah teman-teman buat
     dengan perintah di bawah ini satu-persatu:
     
     $ sudo systemctl daemon-reload
     
     $ sudo systemctl enable redshift-delay.timer
     
     $ sudo systemctl start redshift-delay.timer

Dan selesai! Dengan langkah-langkah di atas, Redshift akan dijalankan otomatis 
setelah uptime sistem mencapai 2 jam. Timer systemd ini memastikan bahwa skrip 
start-redshift.sh dijalankan setelah waktu yang ditentukan.
